üè† The basics

Hydrogen is a utility type CSS framework.

Okay, but what does that mean?

CSS is normally written by applying a class to an element and assigning styles to that class. That class then becomes reusable, so you can make other elements have the same look and feel. This is great, but it becomes a bit unruly once you start having to work out naming conventions for classes, keeping track of which class does what, ensuring consistency between things like color, and trying to ensure that old/unused classes are removed from your code. Not to mention you have to worry about writing code in two places and making sure it aligns.

Utility frameworks in general try to solve these problems by providing you with premade classes that do very specific, predictable things. As a result, once you've learned the syntax for the framework, you almost never find yourself writing CSS - all of the effort goes into applying these pre-existing classes to your HTML. One place, consistent naming, consistent output.

Hydrogen's approach to this is slightly different.

Rather than using classes, Hydrogen uses HTML custom attributes to generate its own language, inspired by CSS. So what does this look like in comparison?

Normal class definition 

In HTML:  
<div class="myPrimaryBackground"></div>

In CSS:
.myPrimaryBackground {
  background-color: blue;
}

Hydrogen definition 

In HTML:  
<div data-h2-bg-color="base(primary)"></div>

By watching for changes and scraping your HTML, Hydrogen understands that you want to apply the color you've set as "primary" to this element's background color, and it does so on the fly. If you decide to change the color to your brand's accent, or even a plain old HEX value, Hydrogen will immediately rebuild your CSS file appropriately.

If you reuse this same declaration, Hydrogen understands that it's already been used and won't duplicate the code, meaning that your final CSS file only ever contains exactly what you've used on your project in the most minimal format possible.

Things get really fun when we start to look at how Hydrogen enables inline manipulation of things that normally wouldn't be available, like media queries, dark mode, and interaction states. This means that you can actually control an element's styles at different viewports right in your HTML, as well as specify styles for dark mode, and even for when an element is hovered or focused.

‚öôÔ∏è Configuration

Following the same theory behind design tokens, Hydrogen offers robust configuration options that allow you to standardize design decisions platform wide. This makes changing those decisions almost instantaneous, as Hydrogen will understand and apply the change automatically the next time it builds your file.

Hydrogen allows you to configure:

  • Media queries
  • Typography settings
  • Colors
  • Gradients
  • Container sizes
  • Font families
  • Radii
  • Shadow
  • Transition settings
  • Whether to include reset styles
  • Variable output

üé® Working with color

Hydrogen allows you to define as many colors as you need and abstract them behind an agnostic term making updating colors extremely easy. The system will automatically generate 3 light and 3 dark color variants for you to use, but you also have the freedom to overwrite these variants if you want something more custom.

Usage examples

data-h2-bg-color="base(primary)"
data-h2-bg-color="base(darker.primary)"
data-h2-bg-color="base(accent.5)"
data-h2-bg-color="base(#FFCC33)"

Hydrogen supports both native CSS grid properties and also comes with a custom flexbox-based grid system for broader support. Because styles are inline, you can control exactly how layouts respond to viewport sizes extremely easily.

CSS grid

<div 
  data-h2-display="base(grid)" 
  data-h2-grid-template-columns="base(1fr) l-tablet(1fr 1fr 1fr)" 
  data-h2-gap="base(x1)">
    <p>Item 1</p>
    <p>Item 2</p>
    <p>Item 3</p>
</div>

üî† Space and type

Space values for margin, padding, height, and width are all calculated based on the current line height setting. This helps maintain a consistent vertical rhythm and ensures that you have additive control over how much space you're applying.

data-h2-height="base(x1)"
data-h2-height="base(x2)"
data-h2-height="base(x5)"

Typography values are automatically generated for you based on your specific settings. This means that heading sizes and their line heights are calculated on the fly, and will adapt with whatever media queries you've set.

‚ö° What's new

1.1.19

Hydrogen used to compile a library of CSS and then match that CSS to the attributes you used. This was pretty slow and extremely limiting, as you could only use the specific settings in your configuration file.

2.0.0

Rather than matching to a library, it now rapidly writes CSS on the fly based on the attribute you give it, meaning it can now accept real CSS values in a lot of cases on top of your settings.

1.1.19

Hydrogen used to only support state management (e.g. adding :h to apply styles to an element's hover effect) at a very basic level.

2.0.0

This has been expanded to include almost all available states and a dark mode modifier so that you can specify dark mode styles right in the attribute as well.

1.1.19

Hydrogen's goal is to make repetitive CSS centralized and easy to write, but for complex CSS like animations and transformations, there was a disconnect, because all of the helpful settings were locked into the system.

2.0.0

You can now enable variable exports in the settings which tells Hydrogen to generate CSS variable, Sass variable, or both files. This means you can use your Hydrogen settings outside of Hydrogen, and they'll always stay in sync.

1.1.19

Typography and whitespace used to be entirely separated and controlled using different settings. Base font size for your project could only be set once, you could only have a single typography scale for headings, and whitespace values were automatically generated from a user-set multiplier value.

2.0.0

Hydrogen now leverages CSS variables to enable a vertical rhythm ecosystem that allows you to set different typography settings and link them to individual media queries. This means you can change your base font size, typography scale, and line height values to better suit different viewports. Whitespace has been completely overhauled to follow the vertical rhythm as well, meaning it creates values for you based on the line height you set for consistent vertical spacing.

1.1.19

Media query settings used too be locked to a specific numeric value and its unit (e.g. 400px) and were limited to screens.

2.0.0

Media queries are now completely customizable, meaning you can create queries for not only certain screen sizes, but ranges of screen sizes and other media like print.

1.1.19

Color modifiers used to be really clunky due to a bracket syntax required for parsing, and only provided you with a single light and dark variant of your color setting. These variants were automated and you couldn't control them.

2.0.0

Colors now use a dot syntax, order of the modifiers is irrelevant, and you now have access to three automated light variations and three automated dark variations of your color settings, all of which can be overwritten if you need them to be.

1.1.19

Despite having 27 different attributes, a lot of commonly used CSS properties were unavailable.

2.0.0

This version adds support for another 14+ attributes, including things like transition and CSS grid support.
