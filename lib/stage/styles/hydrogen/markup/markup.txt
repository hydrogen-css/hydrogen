<h3
  id=""
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Data Attributes?</h3>
<p>Why use <code>data-attributes</code> over <code>class</code>?</p>
<p>There's been a lot of discussion around classes performing better than data attributes, and we took this into consideration when designing Hydrogen. A lot of traditional class practices like BEM are effective, but can result in some really complex, bloated class attributes, which can have impacts on our ability to consume and write code. One of Hydrogen's primary goals is to make the end-user experience AND the developer experience easier, and so we decided to use data attributes for a few reasons:</p>
<ul>
  <li>they provide consistent namespacing</li>
  <li>they provide a context and a value</li>
  <li>their values are flexible, and allow for modifiers</li>
  <li>they allow for complex combinations</li>
  <li>they're easy to manipulate with JavaScript</li>
</ul>
<p>In combination with Hydrogen's ability to generate a custom CSS file that contains only what you've used, the performance trade-offs are worth the improved experience. It was of the utmost importance that Hydrogen considered maintenance and legibility so that markup could be easily understood while retaining good performance and a small footprint.</p>
<h3
  id=""
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Customization?</h3>
<p>Hydrogen offers customization of both style and syntax through a comprehensive JSON configuration file. Include as many media queries, colors, gradients, and more as your project needs to get the job done. The <a href="#configuration" title="Learn more about how to configure Hydrogen.">Configuration and Themes</a> section of the documentation offers more detail about how this file works.</p>
<h3
  id=""
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">File Size?</h3>
<p>When you're ready to publish your project to the world, Hydrogen's build script will process your HTML, React, etc. and find exactly the attributes your project is using. It then compiles a CSS file that is specifically designed for your project. No bloat, no unused classes or CSS. By doing so, Hydrogen's file size drops drastically, improving both page load speeds and the performance of the data-attributes themselves.</p>

              <h3
  id="install"
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Installation</h3>
              <p>First, we'll need to install <a href="https://nodejs.org/en/" title="Learn more about installing Node and NPM." target="_blank">NPM</a> if you don't already have it on your machine. From there, in a command prompt/terminal window, navigate to the root of your project and enter:</p>
              <pre>npm install @hydrogen-design-system/hydrogen.css --save-dev</pre>
              <p>After the installation completes, you can run:</p>
              <pre>npx h2-init</pre>
              <p>The above command will prompt you to enter three folder locations: your markup folder, your scripts folder, and your styles/CSS folder. Simply enter the path from the root of your project (e.g. <code>src/my/markup</code>). Once you've submitted these paths, Hydrogen will automagically create a <code>hydrogen.config.json</code> file in the root of your project. <span data-h2-font-weight="b(600)" data-h2-font-color="b(theme-1)">This file will be where you can customize Hydrogen's settings and theme.</span></p>
              <h3
  id="usage"
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Getting Started</h3>
              <p>In order to start using Hydrogen, you need to enable it on your project. <a href="#commands" title="Learn more about compiling Hydrogen.">After compiling Hydrogen</a>, a complete CSS file will appear in your specified styles folder, so include this file in your page's <code>head</code> element.</p>

  &lt;link rel="stylesheet" href="path/to/hydrogen.css"&gt;

              <p data-h2-margin="b(bottom, s)">Hydrogen contains a handful of basic reset and typography CSS that might affect your existing styles. You can find out more about these styles below if you're trying to determine what might be altering your project's original CSS.</p>

              <h3
  id="syntax"
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Syntax</h3>
              <p>To begin, let's dissect how a Hydrogen attribute works so that we can better understand how to use them.</p>
              <pre>data-h2-PROPERTY="MEDIA:STATE(ARGUMENTS)"</pre>
              <p>Hydrogen attributes have 5 distinct pieces to them:</p>
              <ul>
                <li><code>data-h2-</code>: this is how custom <code>data-attributes</code> are defined in HTML. The <code>h2</code> bit ensures that Hydrogen's attributes are namespaced and won't interfere with other frameworks.</li>
                <li><code>PROPERTY</code>: the property portion is where we specify which CSS property we want to style. <a href="#backgroundColor" title="View a list of properties available in Hydrogen." data-h2-font-color="b:h(theme-1)">A list can be found starting with background color below.</a></li>
                <li><code>MEDIA()</code>: this is how we define at which media query we want our styles to apply at. You can learn more about how to use media queries and which ones are installed by default below.</li>
                <li><code>:STATE</code>: this <strong>optional</strong> value allows you to set Hydrogen styles for a particular state on a handful of Hydrogen properties. <a href="#states" title="Learn more about states in Hydrogen." data-h2-font-color="b:h(theme-1)">You can see a list of of available states and which properties they can be used with below.</a></li>
                <li><code>ARGUMENTS</code>: each property has its own special set of arguments, outlined in the documentation below, that will define how the element is styled.</li>
              </ul>
              <h3
  id="mediaQueries"
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Media Queries</h3>
              <p>Media queries in Hydrogen are designed to allow you to granularly control each property when you need it. Rather than being forced to write unique styles for a particular element, you can directly target that element's property at the exact query you need right in your markup.</p>
              <p>Hydrogen comes with a default set of 4 media queries out of the box:</p>
<pre>
b
s (48em)
m (64em)
l (100em)
</pre>
              <p>Using a media query is simple:</p>
              <pre>data-h2-bg-color="b(black) m(theme-1) l(theme-2)"</pre>
              <p>Renders something like this (try resizing your screen!):</p>
              <div data-h2-bg-color="b(black) m(theme-1) l(theme-2)" data-h2-padding="b(all, l)" data-h2-margin="b(bottom, m)"></div>
              <p>If the built in media queries don't suit your needs, you can <a href="#configMedia" title="Learn more about setting custom media values in your configuration file." data-h2-font-color="b:h(theme-1)">customize them</a> in your <code>hydrogen.config.json</code> file.</p>
              <!-- States -->
              <h3
  id="states"
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">States</h3>
              <p>The state property allows you to modify a media query so that its arguments only apply to a particular element's state. This is handy for manipulating styles on things like <code>:hover</code> so that a link or button behaves a certain way when a user hovers over it with their cursor.</p>
              <p>For example, if we wanted to change the color of a link from red to purple when it's hovered, we'd do the following:</p>

<pre>
&lt;a href=&quot;&quot; title=&quot;&quot; data-h2-font-color=&quot;b(red) b:h(purple)&quot;&gt;My link&lt;/a&gt;
</pre>
              <p>Hydrogen provides options for the following states:</p>
              <ul>
                <li>hover, <code>:h</code></li>
                <li>active, <code>:a</code></li>
                <li>focus, <code>:f</code></li>
                <li>disabled, <code>:d</code></li>
              </ul>
              <p>Because states are intimately tied with the volume of CSS Hydrogen produces, it was important to thoughtfully limit their scope to properties that make sense, rather than allowing their use on every single Hydrogen property. This means that states can be used with the following:</p>
<pre>
bg-color
border
display
font-color
font-style
font-weight
margin
overflow
padding
radius
shadow
visibility
width
</pre>

  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Color and Gradients</h3>
              <p>Hydrogen comes with a set of 5 pre-installed colors, but we know that color is almost always something you're going to want to control. In this section, we want to explain how color is used in the context of property arguments, rather than highlight the colors themselves. <a href="#configThemeColors" title="Learn more about configuring colors.">You can customize the exact palette for your application by following the instructions in the Themes section.</a></p>

              <p><span data-h2-font-color="b(theme-1)" data-h2-font-weight="b(600)">The simplest way to reference a color in a property argument is to simply use its name</span>: <code>data-h2-bg-color="b(purple)"</code>.               <h4
  id="gradients"
  data-h2-font-size="b(h4)"
  data-h2-font-color="b(black)"
  data-h2-font-weight="b(600)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Gradients</h4>
              <p>Hydrogen doesn't ship with any gradients enabled by default, but you can optionally configure them in the <a href="#configThemeGradients" title="Learn more about configuring gradients.">gradients section of your configuration file</a>.</p>
              <p data-h2-font-color="b(theme-1)" data-h2-font-weight="b(600)">Gradients are supported by the following properties:</p>
<pre>
bg-color
</pre>
              <p>When they are enabled, you can reference them in place of a color value by using the following syntax: <code>TYPE-ANGLE[COLORSTOP1][COLORSTOP2]</code>.</p>
              <p>More specifically, radial gradients will not accept an angle value, while linear gradients require them. These angle values are assigned in your configuration file.</p>
              <p data-h2-font-weight="b(600)">Examples:</p>
<pre>
linear-45[purple][blue]
radial[pink][purple]
linear-270[purple][blue][pink]
</pre>
              <h3
  id="whitespace"
  data-h2-font-size="b(h3)"
  data-h2-font-color="b(theme-1)"
  data-h2-font-weight="b(200)"
  data-h2-margin="b(top, none)"
  data-h2-padding="b(top, xl)">Whitespace</h3>
              <p>Whitespace in Hydrogen works within certain constraints. In order to help ensure the flow of whitespace is consistent in your app and you aren't worrying about arbitrary values for margins, padding, and the like, Hydrogen generates a set of whitespace values for you using the <code>whitespaceScale</code> option in your configuration file.</p>
              <p>By default, the <code>whitespaceScale</code> value is set to 1.5, and this means that, starting with <code>1rem</code>, whitespace values are calculated like this:</p>
<pre>
none (0rem)
xxs (0.4444rem)
xs (0.6666rem)
s (1rem)
m (1.5rem)
l (2.25rem)
xl (3.375rem)
xxl (5.0625rem)
</pre>
              <p>These values can be used in any property that has a <code>WHITESPACE</code> argument.</p>

import * as React from "react";
import {
  h2DialogResizeOnViewport,
  h2DialogTrigger,
  h2DialogEnableOpenDialogs,
} from "@hydrogen-design-system/system/dist/import/latest/components/dialog/scripts/dialog";
import { GeneralProps, GeneralBtnProps } from "./utils";

interface DialogContext extends GeneralProps {
  /** The dialogs id. */
  id: string;
}

const DialogContext = React.createContext<DialogContext | undefined>(undefined);

/**
 * This Context hook allows our child components to easily reach
 * into the Tabs context and get the pieces it needs.
 *
 * Bonus: it even makes sure the component is used within a
 * Dialog component!
 */
const useDialogContext = (): Partial<DialogContext> => {
  const context = React.useContext(DialogContext);
  if (!context) {
    throw new Error("This component must be used within a <Dialog> component.");
  }
  return context;
};

const Actions: React.FunctionComponent<GeneralProps> = (props) => {
  useDialogContext(); // Ensures sub-component can only be used within the Dialog component.
  const { className, children, ...rest } = props;
  return (
    <div data-h2-dialog-actions className={className} {...rest}>
      {children}
    </div>
  );
};

const ActionBtn: React.FunctionComponent<GeneralBtnProps & GeneralProps> = (
  props,
) => {
  const { id } = useDialogContext();
  const { buttonStyling, type, onClick, className, children, ...rest } = props;
  const ref = React.useRef(null);
  React.useEffect((): void => {
    h2DialogTrigger(ref.current);
  });
  return (
    <button
      data-h2-dialog-trigger={`${id}`}
      data-h2-button={buttonStyling}
      ref={ref}
      type={type || "button"}
      onClick={onClick}
      className={className}
      {...rest}
    >
      {children}
    </button>
  );
};

const Content: React.FunctionComponent<GeneralProps> = (props) => {
  const { id } = useDialogContext();
  const { className, children, ...rest } = props;
  return (
    <div
      data-h2-dialog-content
      id={`${id}Content`}
      className={className}
      {...rest}
    >
      {children}
    </div>
  );
};

const Header: React.FunctionComponent<GeneralProps> = (props) => {
  useDialogContext(); // Ensures sub-component can only be used within the Dialog component.
  const { className, children, ...rest } = props;
  return (
    <div data-h2-dialog-title className={className} {...rest}>
      {children}
    </div>
  );
};

const Title: React.FunctionComponent<GeneralProps> = (props) => {
  const { id } = useDialogContext();
  const { className, children, ...rest } = props;
  return (
    <h5 data-h2-focus id={`${id}Title`} className={className} {...rest}>
      {children}
    </h5>
  );
};

interface TriggerProps extends GeneralProps, GeneralBtnProps {
  id: string;
}

/** This Trigger component opens the dialog and sits outside the main dialog component */
const Trigger: React.FunctionComponent<TriggerProps> = (props) => {
  const { id, buttonStyling, className, children, ...rest } = props;
  const ref = React.useRef(null);
  React.useEffect((): void => {
    h2DialogTrigger(ref.current);
  });
  return (
    <button
      data-h2-button={buttonStyling}
      ref={ref}
      data-h2-dialog-trigger={`${id}`}
      type="button"
      className={className}
      {...rest}
    >
      {children}
    </button>
  );
};

interface DialogComposition {
  Actions: React.FunctionComponent<GeneralProps>;
  ActionBtn: React.FunctionComponent<GeneralBtnProps>;
  Content: React.FunctionComponent<GeneralProps>;
  Header: React.FunctionComponent<GeneralProps>;
  Title: React.FunctionComponent<GeneralProps>;
  Trigger: React.FunctionComponent<TriggerProps>;
}

const Dialog: React.FunctionComponent<DialogContext> & DialogComposition = (
  props,
) => {
  const { id, className, children, ...rest } = props;
  const ref = React.useRef(null);
  React.useEffect(() => {
    h2DialogResizeOnViewport(ref.current);
    h2DialogEnableOpenDialogs(ref.current);
  });
  return (
    <DialogContext.Provider value={props}>
      <div
        ref={ref}
        aria-hidden="true"
        aria-describedby={`${id}Content`}
        aria-labelledby={`${id}Title`}
        data-h2-dialog={id}
        tabIndex={-1}
        role="dialog"
      >
        <div 
          data-h2-padding="b(all, large)"  
          data-h2-dialog-wrapper className={className} {...rest}>
          {children}
        </div>
      </div>
      <div data-h2-dialog-overlay="black, .9" />
    </DialogContext.Provider>
  );
};

// We expose the children components here, as properties.
// Using the dot notation we explicitly set the composition relationships,
// btw the Dialog component and its sub components.
Dialog.Actions = Actions;
Dialog.ActionBtn = ActionBtn;
Dialog.Content = Content;
Dialog.Header = Header;
Dialog.Title = Title;
Dialog.Trigger = Trigger;

export default Dialog;
